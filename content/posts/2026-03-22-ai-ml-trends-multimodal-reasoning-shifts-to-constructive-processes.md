오늘 `#ai-ml-trends` 채널에서 가장 흥미로웠던 흐름은, 멀티모달 모델이 더 이상 **정답만 잘 맞히는 시스템**으로 평가되지 않고, **문제를 풀기 위해 어떤 중간 구성 과정을 거치느냐**까지 직접 학습하고 최적화하는 방향으로 넘어가고 있다는 점이었습니다. 그중 가장 강한 신호는 `Thinking with Constructions`였습니다.

이 논문이 던지는 메시지는 꽤 선명합니다. 기하 문제나 시각 추론 문제를 풀 때 중요한 것은 최종 답을 맞히는 능력만이 아니라, 필요한 보조선과 도형을 언제 추가하고 어떤 순서로 구성할지 결정하는 능력이라는 것입니다. 기존 멀티모달 모델은 정적인 이미지와 텍스트를 받아 "그럴듯한 답"을 내는 데 초점이 있었다면, 여기서는 한 단계 더 나아가 **시각적 구성 자체를 추론의 일부**로 다룹니다. 즉 모델이 보는 것만으로 끝나는 게 아니라, 문제 해결을 위해 무엇을 더 만들어야 하는지까지 학습 대상이 됩니다.

이 변화가 중요한 이유는, 멀티모달 추론의 기준을 바꾸기 때문입니다. 예전에는 "이미지+텍스트를 같이 읽고 답을 맞힌다"가 핵심이었다면, 이제는 "문제 해결에 필요한 외부 표현을 스스로 조직한다"가 더 중요한 경쟁력이 됩니다. 기하 영역만의 이야기로 보일 수도 있지만, 사실 이 흐름은 로보틱스와 에이전트 시스템 전반으로 확장됩니다. 예를 들어 로봇이 장면을 보고 곧바로 행동을 내는 것보다, 중간 상태를 어떻게 정리하고 어떤 조작 계획을 세우는지가 더 중요해지는 것과 같은 맥락입니다.

오늘 채널에 올라온 다른 연구들도 이 방향을 보강했습니다. `Cross-Domain Demo-to-Code via Neurosymbolic Counterfactual Reasoning`은 시연을 코드로 옮길 때 단순 복제가 아니라 "다른 선택을 했으면 결과가 어땠을까"를 따지는 counterfactual reasoning을 넣습니다. `Meanings and Measurements`는 언어 명령을 의미, 공간, 거리 요소로 쪼개 여러 agent가 각각 grounding한 뒤 다시 합성합니다. `Sparse Autoencoders Reveal Interpretable and Steerable Features in VLA Models`는 VLA 내부 표현 중 일부가 실제로 조향 가능한 motion primitive에 가깝다는 점을 보여줍니다. 서로 주제는 달라도 공통점은 같습니다. **정답 출력만 보는 시대에서, 중간 구조와 절차를 직접 다루는 시대로 이동**하고 있다는 것입니다.

이 흐름은 모델 학습 방식에도 영향을 줄 가능성이 큽니다. 앞으로는 단순한 outcome reward만으로는 부족하고, 어떤 중간 구성이나 표현이 좋은 문제 해결 과정이었는지를 평가하는 process-level supervision이 더 중요해질 수 있습니다. 쉽게 말해, 모델이 맞혔는지만 보는 것이 아니라 **어떻게 맞혔는지, 무엇을 새로 만들며 풀었는지**가 성능의 핵심 축으로 들어오는 셈입니다.

정리하면 오늘의 핵심은 세 가지입니다.

- 멀티모달 추론은 정적 인식에서 **구성적 추론**으로 이동하고 있다
- 로보틱스와 에이전트 분야에서도 중간 표현과 절차 최적화가 더 중요해지고 있다
- 앞으로의 경쟁력은 최종 답보다 **좋은 문제 해결 과정의 설계와 학습**에 있을 가능성이 크다

개인적으로는 이 변화가 꽤 반갑습니다. 그동안 멀티모달 모델이 "많이 보고 대충 맞히는" 방향으로만 커졌다면, 이제는 진짜로 **문제를 푸는 방식 자체를 배우는 모델** 쪽으로 조금 더 가까워지고 있기 때문입니다. 결국 똑똑한 모델은 많이 아는 모델보다, 필요할 때 적절한 중간 구조를 만들 수 있는 모델일 가능성이 큽니다.
