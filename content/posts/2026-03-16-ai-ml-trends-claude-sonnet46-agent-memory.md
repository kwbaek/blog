오늘 #ai-ml-trends 채널에서 주목할 만한 소식이 한꺼번에 쏟아졌습니다. 모델 업데이트, 에이전트 메모리 최적화, 그리고 AI 거버넌스를 둘러싼 논쟁까지 — 방향이 뚜렷합니다. AI가 "잘 만들어진 모델"에서 "실제 운영 가능한 시스템"으로 전환하는 속도가 빠릅니다.

## Claude Sonnet 4.6, 1M 컨텍스트 베타와 함께 출시

가장 큰 뉴스는 Anthropic의 Claude Sonnet 4.6입니다. 코딩·에이전트 플래닝·컴퓨터 사용(computer use)·장기 추론 등 전반을 업그레이드했으며, 1M 토큰 컨텍스트를 베타로 제공합니다. 가격은 Sonnet 4.5와 동일($3/$15 per 1M tokens)로 유지해, 고성능 모델로의 진입장벽을 낮추는 전략이 눈에 띕니다.

특히 컴퓨터 사용 성능 개선을 전면에 내세웠다는 점이 중요합니다. GUI 업무 자동화 경쟁이 단순 API 통합을 넘어 실제 데스크탑 작업 대체로 확장되고 있음을 시사합니다. 개발자가 직접 "클릭하고 타이핑하는" 업무를 에이전트로 위임하는 시대가 본격화됩니다.

## 에이전트 메모리: 토큰 11배 줄이고 성능 유지

에이전트 장기 메모리 최적화 논문이 두 편 주목받았습니다.

첫 번째는 Structured Distillation for Personalized Agent Memory로, raw 대화 히스토리를 전부 저장하는 대신 검색 관련성을 보존하는 방식으로 압축해 토큰 사용량을 최대 11배 줄이면서도 검색 품질을 동등하게 유지합니다. 개인화 에이전트의 비용/확장성 문제를 직접 겨냥한 연구입니다.

두 번째는 LMEB 벤치마크 공개로, 장기 메모리 임베딩 성능을 단순 단기 유사도가 아닌 장기 맥락 누적 환경에서 정량 평가합니다. "RAG 잘 되는 것 같은데 왜 실제 운영하면 이상하지?"라는 막연한 의문에 대한 답을 정량적으로 찾기 시작한 신호입니다.

## AgentRM: 운영체제처럼 에이전트 리소스를 관리한다

멀티 에이전트 파이프라인의 메모리·연산·외부 툴 호출을 운영체제의 리소스 관리 원칙(스케줄링, 우선순위, 격리)으로 통합 관리하는 AgentRM 프레임워크가 제안됐습니다. 기존 에이전트 오케스트레이터가 리소스 경합, 지연, 중복 호출을 제대로 다루지 못한다는 문제를 시스템 레벨에서 해결하려는 접근입니다.

에이전트가 하나일 때는 단순했지만, 실서비스에서 수십 개의 에이전트가 동시에 도구를 호출하는 순간 OS 없는 멀티프로세스 환경처럼 카오스가 됩니다. "에이전트 OS"라는 개념이 추상적인 연구에서 실무 아키텍처 고민으로 내려오고 있습니다.

## Palantir Maven: "클릭 3번"으로 군사 타겟 지정

AIPCon에서 공개된 Palantir Maven Smart System 시연이 큰 파장을 불렀습니다. "Left click, right click, left click"만으로 군사 타격 대상을 지정할 수 있음을 공개 무대에서 처음 보여준 사례입니다. Anthropic의 "인간 감독 없는 자율 무기 사용 거부" 정책과 정면으로 부딪히며, AI 안전·거버넌스 논쟁을 한층 달아오르게 합니다.

기술 자체의 가능성보다 "누가 어떤 맥락에서 사용하는가"에 대한 규범 설계가 얼마나 중요한지를 다시 실감하게 만드는 사례입니다.

## PISmith: RL로 프롬프트 인젝션 방어를 자동 레드팀핑

에이전트 보안 관련 흥미로운 연구도 나왔습니다. PISmith는 LLM 에이전트의 프롬프트 인젝션 방어를 강화학습 기반 레드팀 에이전트로 자동 검증하는 프레임워크입니다. 방어 정책을 고정하지 않고 RL 공격자가 끊임없이 새로운 인젝션 전략을 탐색해 취약점을 노출시키는 구조로, 정적 벤치마크의 한계를 넘으려는 시도입니다.

"출시 전에 한 번 평가했다"가 아니라 "지속적으로 공격하며 방어를 업데이트한다"는 개념은 에이전트 보안의 표준이 되어갈 것입니다.

## 정리: 오늘의 핵심 흐름

오늘 트렌드를 한 문장으로 요약하면 **"모델은 강해졌지만, 운영·메모리·보안·거버넌스라는 인프라 경쟁이 진짜 시작됐다"**입니다. Sonnet 4.6이 1M 컨텍스트를 열었고, 메모리 효율화가 비용 벽을 낮추고 있으며, OS 수준의 리소스 관리가 논의되고 있습니다. 동시에 군사 AI 논쟁은 기술이 가치 판단 없이 배포될 때의 위험을 직시하게 합니다.

앞으로의 AI 경쟁은 "더 똑똑한 모델 만들기"와 "그걸 책임 있게 운영하는 체계 만들기"가 동시에 진행되는 레이스입니다. 어느 한쪽만 잘해서는 안 됩니다.
